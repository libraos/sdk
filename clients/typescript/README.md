# @libraos/client

Framework-agnostic TypeScript client kit for LibraOS. No React/framework
dependencies — consumable by employee-ui, school-ui, or any web/Node/React Native
app. Mirrors the Python SDK split: **generated REST types** + **hand-written**
auth/streaming ergonomics.

## What's in the box

| Layer | Source | Notes |
|---|---|---|
| REST types | `src/_generated/schema.d.ts` | Generated by `openapi-typescript` from `openapi/nova-os-partner.v1.yaml`. Checked in, drift-gated. |
| Typed client | `NovaClient` | Bearer auth + automatic refresh-on-401 over `fetch`. Methods for messages / agents / jobs / documents / sessions / deployment. |
| OIDC helper | `OidcClient` | Auth-Code + PKCE (S256) + refresh. Public-client shape (no secret), transport-agnostic — caller owns the redirect. |
| Streaming | `parseAgUiStream`, `AgUiEvent` | Typed SSE parser + AG-UI event union (mirrors `openapi/ag-ui-events.schema.json`). |

Codegen emits types only — SSE, OIDC, and multipart ergonomics are hand-written
(same as the Python SDK's `_sse.py`).

## Install

> **Pre-release (alpha).** The kit publishes under the `next` dist-tag, so a bare
> `npm install @libraos/client` will **not** resolve until a stable
> `latest` is cut. Until then, install from `@next` (or pin an exact version):

```sh
npm install @libraos/client@next
```

Once a stable version is released, the tag drops:

```sh
npm install @libraos/client   # after a stable `latest` is published
```

## Quick start (partner-minted JWT)

```ts
import { NovaClient } from "@libraos/client";

const nova = new NovaClient({ baseUrl: "https://nova.example", auth: process.env.NOVA_JWT! });

const dep = await nova.getDeployment();          // capabilities read
const msg = await nova.createMessage({           // Anthropic-shaped
  model: "claude-opus-4-8",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello" }],
  metadata: { agent_id: "default" },
});

for await (const ev of nova.streamMessage({ /* ... */ })) {
  if (ev.type === "TEXT_MESSAGE_CONTENT") process.stdout.write(ev.delta);
}
```

## Interactive login (OIDC Auth-Code + PKCE)

The OIDC core is transport-agnostic: the kit builds the authorize URL and exchanges
the code; the consumer opens the browser / `ASWebAuthenticationSession` and hands
the callback URL back. See `docs/oidc-client-flow.md` for the full contract.

```ts
import { NovaClient, OidcClient, MemoryTokenStore } from "@libraos/client";

const oidc = new OidcClient({
  issuer: dep.auth!.issuer!,        // from getDeployment()
  clientId: "nova-os-ui",
  redirectUri: `${location.origin}/oauth/callback`,
  tokenStore: new MemoryTokenStore(), // inject Keychain / secure-store in production
});

const pending = await oidc.login();
window.location.assign(pending.authorizeUrl);
// …on the /oauth/callback route:
await oidc.handleCallback(window.location.href, pending);

const nova = new NovaClient({ baseUrl: "https://nova.example", auth: oidc });
// NovaClient now reads tokens from oidc, refreshing proactively + on 401.
```

`TokenStore` is swappable (`get`/`set`/`clear`) — inject the platform-hardened
store (iOS Keychain, expo-secure-store, HttpOnly cookie via a BFF). `MemoryTokenStore`
is the default for tests/Node.

## Build / develop

```sh
npm install
npm run codegen     # regenerate src/_generated/schema.d.ts from the spec
npm run build       # tsup → ESM + CJS + d.ts
npm run typecheck   # tsc --noEmit
npm test            # vitest
```

The generated schema is committed and drift-checked in CI (same as the Go/Python
clients) — run `npm run codegen` after any spec change.
